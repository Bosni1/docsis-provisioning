==class Form==

Implemented interfaces: IForm

Forms play a role similar to the one controllers play in the MVC model. They serve as an interface between Record objects (the model), and GUI editors (the view). Each Form holds a reference to a Table object.

The heart of a Form is a list of TkVariables, which are used by the GUI as data storage of the editor widgets. Forms trace all changes to the variables and propagate them to the Record objects.

As far as Forms are concerned the Record objects are dumb. The Form does not register any handlers for Record's events, therefore all changes to the "current" record made directly to the Record (not through the traced variable interface), may be lost.

instance variables worth mentioning: Form.table -> reference to a Table object Form.current -> currently edited Record Form.tkvars -> a field name-indexed hash of tk variables

methods: save, reload, delete, new -> database operations on the current record setid (objectid) -> load a row identified by objectid into the current record

Events raised by form objects: before_current_record_change - raised when a request to change the current record
